00 | Hugo工具及基本环境部署

首先，不要依赖别人的部署流程，永远推荐以官方文档为准，以第一手信息为准。

这篇文章仅仅是我个人使用Hugo搭建博客的过程总结。因此关于具体部署细节，我并不会在此过度阐述。

官方文档的指导足够清晰简单，跟着上面做吧，很顺利的。

01 | 选择你中意的皮肤

我之所以选择Hugo，是因为无意中发现了一个网友的博客，使用了Loveit主题，这个主题的风格十分戳我的审美，因此没用多想便选择了Hugo。

除了Loveit外，Hugo还有大量精美的主题，你可以到这看看。

02 | 让主题变成你的样子

了解网站基本结构

2.1 如何编辑你的配置文件？

主题下载之后，需要进一步修改其中的基本元素。再次之前需要缕一缕本地Hugo的文件结构是否正确：

.
├── bin
│   └── hugo.exe
├── Sites
    ├── example
    └── MySite
        ├── content
        └── static
        ├── themes
            └── LoveIt
                ├── config.toml
                └── ......
        ├── themes
        └── ......

如果创建新Site、下载主题一切正常，那么应当得到上述文件结构。其中，需要格外关注LoveIt下的config.toml文件。

在这个文件中，我们可以依次修改网站名称、网站副标题、作者、图片等等。

其中有个要点需要注意，如果你想加入一些图片素材，例如在Mysite主页显示一个xx.png素材，那么请将素材放到./Sites/Mysite/static路径下，并将路径设置成"/xx.png"。

本地使用hugo server –buildDrafts命令启动本地服务器后，config.toml一旦保存修改，网页渲染也会立刻改变，因此很容易就能预览到网页效果。

当然，并非所有配置都是通过该配置文件设置的。更深入的修改可能需要更多前端的基础知识。

03 | 发布第一篇文章

3.1 文章的发布方式

可以通过命令行发布文章，也可以在路径./Sites/Mysite/content下直接创建markdown文档。但需要注意，文档开头需要有“前置参数”，用来告诉Hugo要怎样呈现文章。

3.2 文章的前置参数

官方文档列举的比较全。Chatgpt4列举了几个比较常用的：

Hugo 的前置参数提供了丰富的配置选项来帮助你定义和管理你的内容。下面是一些常用的前置参数及其简要说明和配置方法。注意：Hugo 的前置参数可以使用 YAML、TOML 或 JSON 格式，但出于简洁性，我将使用 TOML 格式为例。

  1. title:

    • 描述：文章或页面的标题。
    • 示例： 
      title = "My Blog Post"

  2. description:

    • 描述：文章或页面的简短描述，通常用于 SEO。
    • 示例： 
      description = "A brief overview of my post."

  3. date:

    • 描述：文章的发布日期。
    • 示例： 
      date = 2021-09-01

  4. lastmod:

    • 描述：内容的最后修改日期。
    • 示例： 
      lastmod = 2021-09-10

  5. draft:

    • 描述：指示文章是否是草稿。
    • 示例： 
      draft = true

  6. author:

    • 描述：文章的作者。
    • 示例： 
      author = "John Doe"

  7. tags 和 categories:

    • 描述：文章的标签和分类。
    • 示例： 
      tags = ["hugo", "blogging", "web"]
categories = ["Web Development", "Tutorials"]

  8. slug:

    • 描述：URL 的自定义部分，替代默认的基于标题生成的 URL。
    • 示例： 
      slug = "custom-url-part"

  9. aliases:

    • 描述：旧的或其他的 URLs，通常用于重定向旧内容到新位置。
    • 示例： 
      aliases = ["/old-url/", "/another-old-url/"]

  10. weight:

    • 描述：在列表中的排序权重，较低的数字排在前面。
    • 示例： 
      weight = 1

  11. resources:

    • 描述：定义页面的资源，如图片或其他媒体。
    • 示例： 
      [[resources]]
src = "**.jpg"
title = "My image title"

这只是 Hugo 前置参数的一个子集。Hugo 非常灵活，还有许多其他的前置参数和配置选项，具体可以根据你的需要和所使用的主题来进行设置。

04 | 敞开大门：将网站部署到Github

官方文档提供了很多种部署方法，我选择将其部署在github上。

Step1：新建一个Github仓库用于存储网页相关文件。

  1. 命名GitHub Pages仓库，这个仓库必须使用特殊的命名格式 <username.github.io>， 是自己的GitHub的用户名。
  2. 勾选Public，设置为公开仓库。
  3. 勾选添加README文件，这会设置 main 分支为仓库的默认主分支，这在后面提交推送博客内容时很重要。

Step2：如果是新手，需要在本地使用Git bash生成SSH密钥

  1. 生成SSH密钥
  2. 将公钥添加到github账户

Step3：在博客的public文件夹下初始化Git

  1. 进入public文件夹下，使用 git init 命令初始化。

  2. 将这个仓库与你的GitHub Pages仓库相关联，使用下面的命令

    git remote add origin git@github.com:<youridname>/<youridname>.github.io.git

git pull --rebase origin main

Step4 更新博客并推送提交

  1. 在本地使用hugo预览并更新博客的网页文件

    hugo server
hugo

  2. 使用git推送更新

    git add .
git commit -m "...(修改的信息)" 
git push origin main
